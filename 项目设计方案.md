# 本地 PDF 价格识别与汇总查询工具项目设计方案

版本：v1.2  
日期：2026-07-08  
适用场景：UK Order 价格 PDF 识别、汇总价格表整合、录单前查询与复核

---

## 1. 项目背景

当前手工录单时需要同时查看大量供应商 PDF 价格表和多份 Excel 汇总价格表。PDF 数量多，价格维度复杂，不同产品的尺寸列、面料等级、版本号和价格类型不完全一致，人工查找和录入容易出现以下问题：

- 需要反复打开 PDF 和 Excel，查找效率低。
- PDF 中价格表版式不统一，肉眼对列容易错。
- 同一产品可能存在 2026.1、2026.2 等多个版本，容易使用旧价格。
- FOB、CIF、FV、EZ-LIVING、Sterling 等价格文件结构不同，难以统一查询。
- 价格数字可能在 PDF 提取时被拆开，例如 `8 65` 实际应为 `865`，`1 ,005` 实际应为 `1005`。
- 手工录单需要快速定位产品、面料、尺寸和价格，并能回看来源。

本项目目标是开发一个可便携运行的 Python 本地工具，让用户可以指定 PDF 文件夹，自动识别、整理、汇总价格数据，并在本地页面上查询、筛选、复核和导出。参考 Excel 文件不是识别 PDF 的必选项，而是当前业务人员正在使用的价格表样本和可选数据源：工具需要兼容这些表格的结构、字段、查询习惯和导出格式，但不能依赖它们才能完成 PDF 识别。

## 3. 项目目标

### 3.1 第一版目标

第一版必须快速可见效果，目标如下：

1. 用户可以在页面上输入或选择 PDF 文件夹路径。
2. 工具可以扫描该文件夹下所有 PDF。
3. 不选择 Excel 时，工具也可以独立完成 PDF 识别和汇总。
4. 用户可选导入现有业务 Excel，用于补充产品、颜色、历史价格和业务查询格式。
5. 工具可以兼容 FOB、EZ-LIVING、Sterling、FV 这些现有业务表的核心功能。
6. 工具可以将 PDF 和可选 Excel 数据统一整理成标准长表。
7. 页面可以展示汇总结果。
8. 页面支持多个筛选项。
9. 页面支持精准匹配和模糊匹配。
10. 页面可以显示异常数据并标红。
11. 页面可以导出整理后的 Excel。
12. 项目交付形态应支持嵌入式 Python 便携运行，减少业务电脑环境依赖。

### 3.2 非目标

第一版不做以下内容：

- 不做复杂权限系统。
- 不做多人在线协作。
- 不做云端数据库。
- 不做完全自动无人工复核的录单系统。
- 不强依赖 OCR。
- 不要求用户每次都提供参考 Excel。
- 不要求业务电脑预装 Python、Anaconda 或开发工具。
- 不做过度复杂的前后端分离。

原因：

当前核心需求是本地快速查价、整理、录单辅助。第一版应优先保证 PDF 主流程可独立运行，同时兼容现有业务 Excel 的导入、查询和导出习惯。

## 4. 推荐技术架构

### 4.1 技术选型

| 模块 | 推荐技术 | 理由 |
|---|---|---|
| 便携运行环境 | Windows Embedded Python | 用户电脑无需安装 Python，复制文件夹即可运行 |
| 本地页面 | Streamlit | 极适合数据工具，表格、筛选、上传/路径输入、复核和导出开发速度快 |
| 表格交互 | Streamlit dataframe / data_editor | 原生支持排序、筛选、编辑、下载和状态刷新 |
| PDF 解析 | pdfplumber | 当前 PDF 可直接提取文字和表格 |
| OCR 备用 | Tesseract 可选插件 | 第一版不内置重 OCR，避免包体过大 |
| Excel 读取 | openpyxl | 读取公式、值、sheet 结构方便 |
| 数据处理 | pandas + SQLite | 与 Streamlit 表格展示天然适配，清洗、宽表转长表效率高 |
| 模糊匹配 | rapidfuzz | 比 fuzzywuzzy 更快，依赖更轻 |
| 本地数据库 | SQLite | 无需安装服务，方便缓存和查询 |
| 导出 Excel | openpyxl | 读取和导出统一，减少依赖 |
| 配置管理 | JSON / TOML | 可直接用标准库读取，减少 PyYAML 依赖 |

技术路线说明：

- Streamlit 作为正式主界面方案，不再仅作为开发验证工具。
- 正式交付版采用嵌入式 Python + Streamlit + SQLite，业务人员通过双击脚本启动。
- 业务人员使用时只需要双击 `启动工具.bat`，无需安装 Python、pip、VS Code 或其他开发环境。
- Streamlit 运行在本机地址，默认 `http://127.0.0.1:8501`，不依赖外网。
- OCR 不进入第一版主包，作为后续可选插件，否则会显著增加包体和部署复杂度。

## 7. PDF 识别设计

### 7.1 PDF 文件名解析

文件名解析规则：

```text
{supplier} Price {product_name} {product_code}_{version} {currency}.pdf
```

示例：

```text
ATH Price Alba 7286_2026.2 USD.pdf
```

解析结果：

```json
{
  "supplier": "ATH",
  "product_name": "Alba",
  "product_code": "7286",
  "version": "2026.2",
  "currency": "USD"
}
```

特殊情况：

```text
ATH Price Atlantis 7112-7162_2026.2 USD.pdf
ATH Price Winslet 7205-7209_2026.2 USD.pdf
HYD Price Lounge Chairs_2026.1 USD.pdf
```

处理策略：

- `7112-7162` 保留原始 `product_code_raw`。
- 同时拆分为多个 product code：`7112`、`7162`。
- 如果文件名没有明显款号，例如 `Lounge Chairs`，则 product_code 为空并标记复核。

### 7.2 PDF 表格提取

使用 `pdfplumber`：

```python
with pdfplumber.open(pdf_path) as pdf:
    for page_index, page in enumerate(pdf.pages, start=1):
        tables = page.extract_tables()
```

每个表格保留：

```text
source_file
page
table_index
raw_rows
```

### 7.4 价格清洗

PDF 中常见价格问题：

| 原始值 | 清洗结果 |
|---|---:|
| `8 65` | 865 |
| `6 90` | 690 |
| `1 ,005` | 1005 |
| `1 ,100` | 1100 |
| `5 40` | 540 |
| `-` | 空 |
| 空白 | 空 |

清洗函数设计：

```python
def clean_price(value: str) -> tuple[float | None, str | None]:
    """
    返回: (price, issue)
    issue 为空表示正常；不为空表示需要复核原因。
    """
```

清洗规则：

1. 去除换行、空格、货币符号。
2. 将 `1 ,005` 修正为 `1005`。
3. 将 `8 65` 修正为 `865`。
4. 将 `-`、`N/A`、空白识别为空。
5. 转换失败则 `needs_review = 1`。
6. 价格小于 100 或大于 5000，标记复核。

## 8. Excel 导入设计

### 8.0 Excel 输入原则

Excel 文件在本项目中是可选输入，不是 PDF 识别的前置条件。

页面行为：

```text
只选择 PDF 文件夹：
  扫描 PDF -> 识别价格 -> 生成汇总 -> 页面筛选 -> 导出

选择 PDF 文件夹 + 业务 Excel：
  扫描 PDF -> 导入 Excel -> 合并汇总 -> 对比差异 -> 页面筛选 -> 导出

只选择业务 Excel：
  导入 Excel -> 生成统一价格库 -> 页面筛选 -> 导出
```

设计原则：

- 工具内置业务 Excel 解析配置，用户不需要手动指定每一列含义。
- Excel 导入失败不应阻断 PDF 主流程。
- 同一批次中 PDF 和 Excel 的数据都写入 `prices`，通过 `source_type`、`source_role`、`source_profile` 区分。
- Excel 的公式、原始 sheet、原始行列号应尽量保留，便于追溯。
- 如果 Excel 只是作为模板参考，用户可以不导入。

### 8.1 长表 Excel 导入

适用于：

```text
FOB_Price_List_2026_July_NEW.xlsx
EZ-LIVING_Price_List_2026_July_New_v5_FIXED.xlsx
```

读取逻辑：

1. 优先寻找 `All Prices` sheet。
2. 读取标准字段。
3. 生成 `lookup_key`。
4. 写入 `prices`。
5. 读取 `Colour Reference` 写入 `colour_reference`。
6. 读取 `Products` 写入 `products`。
7. 如果存在 `Quick Lookup`，解析其查询字段和公式逻辑，转换为页面查询配置，不强制复制 Excel 公式。

字段映射：

| Excel 字段 | 系统字段 |
|---|---|
| Product Code | product_code |
| Product Name | product_name |
| Section | section |
| Tier | tier |
| Cover Range | cover_range |
| Cover Type | cover_type |
| Size | size |
| Price (USD) | price |
| Lookup Key | lookup_key |

业务功能兼容：

| 现有 Excel 功能 | 工具中对应设计 |
|---|---|
| Quick Lookup 按产品、Cover、Size 查价 | 汇总查询页面的精准查询和组合筛选 |
| All Prices 长表 | `prices` 主表 |
| Colour Reference | `colour_reference` 表和颜色/面料反查 |
| Products | `products` 表和产品下拉筛选 |
| Lookup Key | 系统自动生成统一 `lookup_key` |

### 8.2 宽表 Excel 导入

适用于：

```text
Sterling Price List CIF 01072026.xlsx
FV Prices 2026.xlsx
```

读取逻辑：

1. 遍历所有 sheet。
2. 从 sheet 名或表格标题识别产品代码和产品名。
3. 自动定位表头行。
4. 识别尺寸列。
5. 将宽表转成长表。
6. 保留原始 sheet、行号、列号。
7. 对公式单元格同时读取显示值和公式文本。
8. 对宽表中的备注列、脚注列保留到 `review_reason` 或原始导入表。

宽表转长表示例：

原始：

| FABRIC/LEATHER | COVER | 1 SEATER | 2 SEATER | FOOTSTOOL |
|---|---|---:|---:|---:|
| FABRIC | EDEN / LAVISH / CHILTERN | 355 | 515 | 200 |

转换：

| section | cover_range | size | price |
|---|---|---|---:|
| FABRIC | EDEN / LAVISH / CHILTERN | 1S | 355 |
| FABRIC | EDEN / LAVISH / CHILTERN | 2S | 515 |
| FABRIC | EDEN / LAVISH / CHILTERN | FOOTSTOOL | 200 |

### 9.4 精准匹配与模糊匹配设计

#### 精准匹配

适用场景：

- 用户明确知道产品代码。
- 用户明确知道尺寸。
- 用户明确知道 Tier。

规则：

```text

输入 CHILTERN，只匹配 cover_range = CHILTERN
输入 3S，只匹配 size = 3S
```

#### 模糊匹配

适用场景：

- 用户只记得部分产品名。
- PDF 或 Excel 中有拼写差异。
- 面料名称中包含多个组合词。

规则：

```text
输入 chiltern，可匹配：
- CHILTERN
- EDEN / LAVISH / CHILTERN
- CHLTERN

输入 atlantis，可匹配：
- Atlantis
- 7112 Atlantis Scatter Back / 7162 Atlantis Formal Back

输入 foot，可匹配：
- FOOTSTOOL
- 7163 Denali / 7162 Lima Footstools
- 7008 Footstool
```

技术实现：

```python
from rapidfuzz import fuzz

score = fuzz.partial_ratio(query.lower(), target.lower())
matched = score >= threshold
```

建议阈值：

| 匹配对象 | 阈值 |
|---|---:|
| product_code | 100 |
| product_name | 70 |
| cover_range | 70 |
| size | 85 |
| source_file | 60 |

页面应允许用户调整模糊匹配阈值。

### 17.6 Streamlit 页面状态与服务函数设计

Streamlit 不需要单独维护一套 HTTP API。页面直接调用 `src/` 中的服务函数，状态放在 `st.session_state`，大数据结果落到 SQLite。

    ...

## 20. 风险与应对

### 20.1 PDF 表格版式不统一

风险：

不同产品 PDF 表格结构不同，单一解析规则不能覆盖所有文件。

应对：

- 第一版先覆盖主流版式。
- 无法识别的行进入异常复核。
- 原始表格完整保存，方便回看。
- 后续按产品增加模板规则。

### 20.2 PDF 价格数字被拆开

风险：

`8 65`、`1 ,005` 等值会造成价格错误。

应对：

- 建立价格清洗规则。
- 清洗前后都保存。
- 对异常价格范围标记复核。

### 20.3 同一产品多个版本

风险：

可能误用旧版本价格。

应对：

- 解析 version。
- 页面提供 `只看最新版本`。
- 多版本冲突进入复核。

### 20.4 Excel 公式与显示值差异

风险：

读取公式时可能拿到公式文本而不是计算值。

应对：

- 使用 `data_only=True` 读取显示值。
- 同时使用 `data_only=False` 保留公式。
- 导出时保留公式来源字段。

### 20.5 模糊匹配误匹配

风险：

模糊匹配可能返回不相关结果。

应对：

- 提供匹配阈值。
- 显示匹配分数。
- 对产品代码默认只做精准匹配。

### 20.6 嵌入式 Python 依赖打包问题

风险：

嵌入式 Python 默认不等同于完整 Python 安装环境。Streamlit 及其依赖包数量较多，可能涉及路径、DLL、缓存目录、浏览器自动打开和隐藏依赖问题。

应对：

- 构建机使用固定 Python 版本，例如 Python 3.11.x 64-bit。
- 所有依赖安装到便携包内的 `python/Lib/site-packages`。
- 构建后必须在未安装 Python 的 Windows 电脑上测试。
- 便携版默认包含 Streamlit 和 pandas，因为它们是页面和表格处理核心。
- 便携版不默认包含 OCR，降低依赖复杂度。
- 日志写入 `logs/`，启动失败时能定位缺失模块或 DLL。
- `启动工具.bat` 中显式设置 `PYTHONHOME` 和 `PYTHONPATH`。
- `.streamlit/config.toml` 中关闭文件监听，避免便携环境下 watcher 报错。

本项目最可落地的路线是：

1. 先以 PDF 文件夹识别作为主流程，确保不依赖任何 Excel 也能扫描、识别、汇总、筛选和导出。
2. 以 `FOB_Price_List_2026_July_NEW.xlsx` 的 `All Prices` 作为标准数据结构参考，但不把该文件作为运行必选项。
3. 将 PDF、Sterling、FV、EZ-LIVING 等不同来源在系统内部统一转换为长表，通过 `source_type`、`source_role`、`source_profile` 区分来源。
4. 页面重点做文件夹选择、可选业务 Excel 导入、汇总展示、多条件筛选、精准/模糊匹配、异常复核和 Excel 导出。
5. 正式交付版采用嵌入式 Python + Streamlit + SQLite，业务人员双击启动即可使用。
6. 不追求一开始完全自动识别所有复杂版式，而是保留异常复核页面，做到可控、可查、可追溯。

这样既能最快看到效果，也能保证工具可移植、可维护，并兼容现有业务人员正在使用的 Excel 表格习惯。
